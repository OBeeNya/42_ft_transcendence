Un utilisateur essaye de rejoindre un channel :

- l'utilisateur n'est pas membre du channel et ce dernier est public --> il le rejoint

- le channel est privé --> l'utilisateur a besoin d'une invitation 
  ET/OU le propriétaire du channel doit approuver la demande pour rejoindre

- le channel est protégé par mot de passe : l'utilisateur doit fournir le bon mdp pour rejoindre

- l'utilisateur est deja membre du channel

- Le channel est plein

- L'utilisateur a ete bloque par le propriétaire du channel ou
  par l'administrateur

- L'utilisateur a été banni du channel

Fonctionnement du blocage dans mon code :

BACKEND

1) schema.prisma : lorsqu'un utilisateur bloque un autre utilisateur, une instance
de la table UserBlock est créée, celle ci contenant le blockerId et le blockedId.
Dans User, userBlocks est une liste de toutes les instances de UserBlock où l'utilisateur
actuel à initié le blocage. Donc userBlocks contient tous les utilisateus bloqués
par l'utilisateur actuel.
L'inverse pour blockedUsers.

2) blockage.gateway .service : handleBlockUser() attends un message "blockUser" de la part
du client contenant l'id de l'utilisateur qui bloque (blockerId) et l'id de l'utilisateur
bloqué (blockedId). S'il existe et qu'il n'est pas déjà bloqué, elle utilise blockUser() de
blockage.service pour créer une nouvelle entrée dans la table UserBlock contenant les id
appropriés.

Blocage

Ce que dis la soutenance : un utilisateur qui a bloqué un autre utilisateur ne
doit plus recevoir ses messages, ni voir ses messages dans les channels.

1) L'utilisateur clique sur "Block" avec une page vierge --> l'utilisateur bloqué
clique sur "Direct Message mais ça ne fait rien".

2) L'utilisateur clique sur "Block" avec une conversation privée ouverte -->
il rentre des strings mais elles ne s'envoient pas.
